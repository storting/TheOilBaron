Концепция

Бизнес-кликер про фермера, который случайно находит нефть на своем участке. Игрок развивает нефтяной бизнес, прокачивает оборудование и персонажа, продает нефть компаниям и расширяет свою империю.

Целевая аудитория

  • Возраст: 16-35 лет
  • Любители казуальных игр в транспорте
  • Игроки, ценящие прогресс и развитие
  • Аудитория мобильных инкрементальных игр

Уникальное торговое предложение (USP)

  • Стиль графики в духе "Футурамы" и "Гриффинов"
  • Глубокая система прокачки (насос, персонаж, окружение)
  • Динамическая экономика с меняющимися ценами
  • Юмористический подход к теме нефтяного бизнеса

Основной геймплей

Core Loop

  • Нажимать на экран → Добывать нефть
  • Продавать нефть → Получать деньги
  • Прокачивать насос/персонажа → Увеличивать эффективность
  • Переходить на новую локацию → Расширять бизнес

    • Ключевые механики

    • Активная добыча: Тап по насосу для мгновенной добычи
    • Пассивный доход: Автоматическая добыча в оффлайне
    • Торговая система: Продажа нефти компаниям по динамическим ценам
    • Система прокачки: 3 уровня улучшений (насос, персонаж, окружение)
    • Смена локаций: Новые земли с большими запасами нефти

    Прогрессия игрока

    Уровень 1: Небольшая ферма, ручная добыча
    Уровень 5: Автоматизация, первые улучшения
    Уровень 15: Собственный нефтеперерабатывающий завод
    Уровень 30: Нефтяной магнат, международный бизнес

    Ценообразование

    • Цены у компаний меняются каждые 2-4 часа
    • Факторы: спрос, мировые события, политическая ситуация
    • Каждая компания покупает ограниченное количество нефти
    • Игрок должен выбирать лучшие предложения

    Code Review Guidelines

    Обязательные требования

    • Именование переменных: Понятные имена на английском
    • Комментарии: Сложная логика должна быть объяснена
    • Оптимизация: Избегать тяжелых операций в Update()
    • Архитектура: Новые фичи следуют существующей структуре

    Проверка перед коммитом

    • Код компилируется без ошибок
    • Нет критических багов в тестировании
    • Добавлены/обновлены комментарии к новому коду
    • Соблюдены соглашения по именованию

    Частые проблемы

    Выносить в константы
